Tips and Tricks in LaTex

December 2, 2021

This post continuously updating.#

This post includes some tips for writing an essay individually or in a group using LaTex.

The best practice posts and other recommendations in my weblog are local best practice, so better ways probably exist and I don’t know about them until now, please write about them and share them with me. Such posts aim to share my way of doing this procedure without repeating it.

Firstly, I introduce some tools:

Overleaf is a good tool to share and work on a paper in a group, but I prefer to use Git, instead.

MathPix is a fantastic tool to take pictures/scans for converting them into LaTex formula.

stargazer is an incredible R package to generate LaTex regression tables. It’s easy to use and worth trying.

tablesgenerator.com is a useful site to generate simple tables with ease.

Now some tips for writing documents using LaTex:

If it’s enough large paper, it’s possible to split the paper into multi sections, write each section in a different file, then add them into the main Tex file using \input

|- intro.tex
|- section-1.tex
|- section-2.tex
|- main.tex

and main.tex contains something like below:

\input{into}
\input{section-1}
\input{section-2}

 

Everybody appreciate helpful comments, please write them. It’s a good idea to keep them even at the end.  

For cross-referencing equations, sections always use label do not use the logical number in your mind, because the layout and ordering probably change throughout the time. Use meaningful names for labels, using a combination of category’s name and equation/subject’s name is going to be helpful. for example:

\begin{equation}
 e=mc^2\label{eqn:mass–energy}
\end{equation}

I use eqn for equation, lem for lemma, and thm for theorem as category names. You can use your preferred names in this situation.  

Adding citations in the main file is a bad practice, for citing and referencing BibTex is a good choice. If you working in a specific area, you may enjoy having a big .bib file, but you should keep it updated during the time. For instance, pre-print papers will publish in a journal and their information will be changed.  

Assume you have to deal with a big equation, always it’s possible to break into multi sections, hopefully, any big term has a dedicated line.  

Using macros can be helpful a lot. I have a file including all frequently used macros, mostly calligraphy letters and combination of left-to-right operator and other operators, It’s handy for writing Persian docs or any left-to-right language. I just add my.sty file into the project and boom! But more importantly, defining variables with macros, including basic variabls like X,Y or an algorithm name you should decide later, will be helpful to change them in the future.
